Modul 02 · Claude Mastery Curriculum

Memory verstehen.

Was Claude wirklich speichert, was er vergisst, und wie du die drei Memory-Typen gezielt einsetzt — damit Claude dich kennt ohne alles jedes Mal neu erklären zu müssen.

Reading: 30 Min Hands-on: 30 Min Schwierigkeit: Einsteiger
Lernziele

📋 Was du nach diesem Modul kannst

Ziele
  • Erklären warum Claude nach jedem Chat "alles vergisst" — und was stimmt daran nicht
  • Unterscheiden zwischen Global Memory, Project Memory und Context Window
  • Editieren und löschen was Claude über dich gespeichert hat
  • Optimieren deine bestehenden 21 Memory-Edits für bessere Ergebnisse
  • Verstehen warum das Turbo-Projects Projekt und das Mastery-Projekt getrennte Memories haben
Sektion 1

🧠 Mental Model — Das Gedächtnis-Paradox

Die häufigste Aussage über Claude: "Claude vergisst alles nach jedem Chat."

Das stimmt — und stimmt nicht. Hier ist was wirklich passiert:

WAS DU GLAUBST: Chat 1 Chat 2 Chat 3 ────── ────── ────── "Ich bin "Wer bist "Hallo?" Schadi" du nochmal?" → Claude vergisst alles zwischen Chats ❌ (nur teilweise wahr) ───────────────────────────────────────────────────────────── WAS WIRKLICH PASSIERT: ┌─────────────────────────────────────────────────────┐ │ GLOBAL MEMORY (persistent, chat-übergreifend) │ │ "Schadi = CEO Grelle Forelle, Wien, Du-Form" │ │ Wird in JEDEN Chat geladen, automatisch │ └────────────────────────┬────────────────────────────┘ │ ┌──────────────┴──────────────┐ ▼ ▼ ┌──────────────────┐ ┌──────────────────────┐ │ PROJECT MEMORY │ │ PROJECT MEMORY │ │ Turbo Projects │ │ Claude Mastery │ │ (Hub, MCPs, │ │ (Module, Lernplan, │ │ Operations) │ │ Format-Specs) │ └──────────────────┘ └──────────────────────┘ │ │ ▼ ▼ ┌──────────────────────────────────────────────────┐ │ CONTEXT WINDOW (pro Chat) │ │ Das aktuelle Gespräch — verschwindet wenn │ │ der Chat endet oder das Fenster voll ist │ └──────────────────────────────────────────────────┘

Die Wahrheit: Claude vergisst den Inhalt einzelner Chats — aber nicht was du als Memory gespeichert hast. Diese drei Schichten funktionieren zusammen.

Sektion 2

📦 Die drei Memory-Typen

Global Global Memory — Dein dauerhaftes Profil

Gilt in jedem Chat, in jedem Projekt. Claude lädt es automatisch am Start jeder Konversation. Das ist das Fundament — wer du bist, wie du kommunizierst, was deine Präferenzen sind.

Dein Beispiel: Deine 21 Memory-Edits (Du-Form, kein Fluff, Token-Count am Ende, Voice-to-Text, etc.) sind Global Memory.

Project Project Memory — Projekt-spezifischer Kontext

Gilt nur innerhalb eines Projekts. Wenn du im Turbo-Projects Projekt bist, hat Claude Zugriff auf die Project Instructions, die hochgeladenen Files, und den Kontext dieses Projekts — aber nicht auf die des Mastery-Projekts.

Dein Beispiel: Im Turbo-Projects Projekt kennt Claude Hub-Struktur, MCPs, Skill-Definitionen. Im Mastery-Projekt kennt Claude den Lernplan, Module, Format-Specs.

Context Context Window — Das aktuelle Gespräch

Alles was in diesem Chat gesagt wurde. Temporär — endet wenn der Chat endet oder das Fenster (200k Token) voll ist. Claude "erinnert" sich an alles was im aktuellen Chat steht, auch wenn du es vor 50 Nachrichten gesagt hast.

Wichtig: Beim Start eines neuen Chats beginnt das Context Window bei null — nur Global + Project Memory bleiben.

Sektion 3

🌐 Global Memory — Dein dauerhaftes Profil

Was Claude über dich gespeichert hat

In Settings → Memory siehst du was Claude global gespeichert hat. Das sind automatisch generierte Zusammenfassungen aus deinen Gesprächen — plus was du explizit als Edits hinzugefügt hast.

💡 Deine Situation

Du hast 21 aktive Memory-Edits. Das ist die Summe aus User Preferences (deine Profile-Settings) und was Claude automatisch aus Gesprächen extrahiert hat. Diese 21 Einträge werden in jeden deiner Chats geladen.

Was macht gutes Global Memory aus?

Beispiel · Global Memory in Action
Schreib mir einen kurzen Text für die Website.
Claude weiß (aus Global Memory): → Du-Form, kein Fluff → Du bist CEO Grelle Forelle, Wien → Direct, business, keine langen Intros → Token-Count am Ende Ergebnis: Antwort ohne "Gerne!", ohne Intro-Geschwätz, direkt auf den Punkt — ohne dass du es jedes Mal sagst.
Sektion 4

📁 Project Memory — Projekt-spezifischer Kontext

Wie Project Memory funktioniert

Jedes Claude-Projekt hat seinen eigenen Kontext-Raum. Das besteht aus drei Elementen:

  1. Project Instructions — Was du in "Edit Project → Instructions" geschrieben hast. Wird in jeden Chat in diesem Projekt geladen. (Wie dieser System-Prompt hier.)
  2. Project Files — Files die du ins Projekt hochgeladen hast. Claude kann sie durchsuchen und referenzieren.
  3. Claude's Memory (project-scoped) — Was Claude automatisch aus Chats in diesem Projekt extrahiert hat — aber nur für dieses Projekt.
TURBO PROJECTS PROJEKT CLAUDE MASTERY PROJEKT ──────────────────────── ──────────────────────── Project Instructions: Project Instructions: - Hub-Dev Kontext - Learn-Only Modus - Skill-Definitionen - Module-Format LOCKED - MCP-Setup - No operative Tasks Project Files: Project Files: - SKILL.md files - Lernplan_Index.md - Hub-Code - Business_Context_Lite.md - Business Context - Module (Modul_07, etc.) Memory (project-scoped): Memory (project-scoped): - "Hub hat Sections X,Y,Z" - "18 Module, Status 0/18" - "Aktive MCPs: 11" - "Format-Ref = Modul 7" ↓ SICHTBAR HIER ↓ SICHTBAR HIER ✅ Turbo Chats ✅ Mastery Chats ❌ Mastery Chats ❌ Turbo Chats

Warum die Trennung? Das ist kein Bug — das ist Design. Operative ToDos aus Turbo Projects sollen nicht in dein Lern-Projekt bluten. Und Modul-Specs aus Mastery sollen nicht in Daily Operations stören.

⚠️ Wichtig

Project Files sind read-only Referenz — Claude liest sie, ändert sie aber nicht. Wenn du ein File updatest musst du es manuell ersetzen.

Sektion 5

💬 Context Window — Das aktuelle Gespräch

Was ist das Context Window?

Das Context Window ist der "Arbeitsspeicher" eines Chats. Alles was in diesem Gespräch steht — deine Nachrichten, Claudes Antworten, hochgeladene Files, Tool-Ergebnisse — das ist im Context Window.

Bei Claude: 200.000 Token = ca. 150.000 Wörter = ein sehr langer Roman. Das reicht für lange, komplexe Gespräche.

Chat Start Chat Ende │ │ ▼ ▼ ┌───────────────────────────────────────────────────┐ │ Context Window (200k Token) │ │ │ │ [Global Memory geladen] │ │ [Project Memory geladen] │ │ Nachricht 1: "Bau mir ein Factsheet für..." │ │ Antwort 1: [Factsheet] │ │ Nachricht 2: "Ändere den Zeitplan auf..." │ │ Antwort 2: [Updated Factsheet] │ │ ... │ │ Nachricht N: [bis 200k Token voll] │ │ │ └───────────────────────────────────────────────────┘ │ ▼ Chat endet → Context Window geleert Global Memory bleibt ✅ Project Memory bleibt ✅ Gesprächsinhalt weg ❌

Wann ist das Context Window relevant für dich?

Beispiel · Hub-Dev Session
Ich arbeite 3 Stunden am Hub-Code in einem Chat. Am Ende: "Fass nochmal zusammen was wir heute gemacht haben."
Claude hat alles im Context Window → kann perfekte Zusammenfassung geben. Nächster Tag, neuer Chat → Context weg. Deshalb: Session Handoff am Ende jeder Session → die wichtigsten Infos landen als File in Project Files.
Sektion 6

✏️ Memory editieren und löschen

Global Memory editieren

  1. Geh zu Settings → Memory in claude.ai
  2. Du siehst alle gespeicherten Memory-Einträge
  3. Klick auf einen Eintrag → Edit oder Delete
  4. Oder sag direkt im Chat: "Bitte merke dir dass ich ab jetzt immer auf Englisch antworten will"
💡 Direkt sagen

Du kannst Claude im Chat direkt sagen was er sich merken soll: "Merke dir: Matthäus ist unser CFO und bevorzugt Zahlen in Tabellenform" — Claude speichert das automatisch in Global Memory.

Was du löschen solltest

Geh durch deine 21 Memory-Edits und lösche:

  • Einträge die veraltet sind (alte Projekte, Personen die nicht mehr relevant sind)
  • Zu spezifische Einzel-Infos die das Memory aufblähen
  • Widersprüchliche Einträge (wenn du früher "immer formal" und später "casual" gesagt hast)

User Preferences vs. Memory-Edits

Es gibt zwei Orte für globale Präferenzen:

OrtWas reinWie ändern
Settings → Profile → User PreferencesPermanente Verhaltens-Regeln (Kommunikationsstil, Format)Settings → Profile bearbeiten
Settings → MemoryAuto-generierte + explizit gesetzte Infos über dichMemory Settings → einzeln editieren/löschen
Sektion 7

🚨 Anti-Patterns — Was du vermeiden sollst

Anti-Pattern #1
Alles im Chat immer wieder erklären

Du erklärst in jedem Chat neu wer du bist, was Grelle Forelle ist, wie du kommunizieren willst. Das gehört ein einziges Mal in Global Memory oder User Preferences — dann nie wieder.

Anti-Pattern #2
Operative Details in Global Memory

Du speicherst sehr spezifische operative Infos (Event-Daten, aktuelle Preise, einzelne Entscheidungen) in Global Memory. Das gehört in Project Files oder Context — nicht in ein dauerhaftes Profil das für jeden Chat gilt.

Anti-Pattern #3
Nie Memory aufräumen

Memory akkumuliert über Zeit. Veraltete Einträge, Widersprüche, redundante Infos — das verwirrt Claude. Alle 2-3 Monate: Memory-Audit machen, aufräumen.

Anti-Pattern #4
Glauben Projekt-Memory ist global

Du sagst im Mastery-Projekt "merke dir dass der Hub Section X hat" — und wunderst dich warum Claude das im Turbo-Chat nicht weiß. Project Memory ist projekt-isoliert. Globales geht in Global Memory.

Hands-on Übung

🛠️ Memory-Audit — 30 Minuten

Übung · Dein Memory aufräumen
3 Aufgaben
  1. Global Memory checken: Geh zu Settings → Memory. Lies alle 21 Einträge durch. Markiere mental: welche sind veraltet? Welche sind zu spezifisch? Lösche mindestens 2 die nicht mehr passen.
  2. Memory setzen im Chat: Schreib in diesem Chat: "Merke dir: [eine echte Info über dich oder deinen Workflow die noch fehlt]". Schau danach in Settings → Memory ob es gespeichert wurde.
  3. Project Memory verstehen: Geh in dein Turbo-Projects Projekt und schau dir die Project Instructions an. Vergleiche mit den Instructions hier im Mastery-Projekt. Siehst du den Unterschied im Kontext?
