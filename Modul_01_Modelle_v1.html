Modul 01 · Claude Mastery Curriculum

Modelle verstehen.

Opus, Sonnet, Haiku — wann nutzt du welches, was kostet es, und warum macht die Wahl einen echten Unterschied in deinem Workflow.

Reading: 30 Min Hands-on: 15 Min Schwierigkeit: Einsteiger
Lernziele

📋 Was du nach diesem Modul kannst

Ziele
  • Erklären was den Unterschied zwischen Opus, Sonnet und Haiku ausmacht
  • Entscheiden in 10 Sekunden welches Modell für welchen Task richtig ist
  • Wechseln zwischen Modellen im Chat — und wissen wann das sinnvoll ist
  • Vermeiden die häufigsten Fehler (Opus für alles = teuer + langsam)
  • Anwenden die richtige Modell-Wahl auf deine Grelle Forelle Workflows
Sektion 1

🧠 Mental Model — Drei Mitarbeiter, ein Büro

Stell dir vor du hast drei Mitarbeiter. Alle drei können schreiben, analysieren, planen. Aber sie haben unterschiedliche Stärken, unterschiedliche Geschwindigkeit, und unterschiedliche Kosten.

DEIN TASK │ ▼ ┌─────────────────────────────────────────────────────┐ │ │ │ OPUS 4 SONNET 4.6 HAIKU 4.5 │ │ ───────── ────────── ───────── │ │ Der Senior Der Allrounder Der Runner │ │ Consultant ✅ Default ⚡ Blitz │ │ │ │ Langsam Mittel Schnell │ │ Teuer Moderat Günstig │ │ Tiefste Analyse Stark + Effizient Quick Tasks │ │ │ └─────────────────────────────────────────────────────┘ │ │ │ ▼ ▼ ▼ Hub-Code bauen Sandra-Mail "Was kostet Strategie-Analyse Factsheet erstellen Heineken?" Komplexe Prompts Daily Briefing Kurze Checks

Der Fehler den die meisten machen: Opus für alles. Das ist so als würdest du den Senior Consultant beauftragen, dir einen Kaffee zu holen. Er kann's — aber es ist Verschwendung.

💡 Key Insight

Für 80% deiner täglichen Tasks ist Sonnet die richtige Wahl. Opus brauchst du nur wenn Sonnet wirklich an seine Grenzen stößt. Haiku für alles was unter 30 Sekunden erledigt sein muss.

Sektion 2

🤖 Die drei Modelle im Detail

Opus
claude-opus-4-6 · Flagship
Stärkstes Reasoning. Komplexe mehrstufige Aufgaben. Lange Kontexte mit vielen Abhängigkeiten. Der einzige der wirklich tief denkt und unerwartete Verbindungen zieht.
Einsatz wenn: Code-Architektur, Strategie-Analyse, komplexe Workflows bauen, wenn Sonnet zweimal falsch liegt.
Sonnet
claude-sonnet-4-6 · Default ✅
Bestes Preis-Leistungs-Verhältnis. Stark genug für 80% aller Tasks. Schneller als Opus. Das Arbeitspferd deines Workflows — zuverlässig, intelligent, effizient.
Einsatz wenn: Emails, Factsheets, Analysen, Content, Planung, Code-Fixes, Daily Operations.
Haiku
claude-haiku-4-5 · Speed
Blitzschnell. Günstigste Option. Für einfache, klare Tasks die keine Tiefe brauchen. Wenn du in 5 Sekunden eine Antwort willst und die Frage simpel ist.
Einsatz wenn: Kurze Fakten, einfache Übersetzungen, schnelle Checks, Routing-Entscheidungen in Automationen.

⚠️ Wichtig zu wissen

Alle drei Modelle haben das gleiche Kontext-Fenster (200k Token) und die gleichen MCP-Integrationen. Der Unterschied liegt im Reasoning, nicht im Zugriff auf Tools.

Sektion 3

🎯 Wann was nutzen — Grelle Forelle Kontext

Deine häufigsten Tasks, richtig zugeordnet

Beispiel 1 · Sandra-Mail (Vermieterin)
Schreib eine formelle Mail an Sandra wegen der Heizungsreparatur.
→ SONNET ist richtig. Kein Opus nötig. Sandra-Mails brauchen Tonalität + Struktur, keine Deep-Reasoning-Kapazität. Sonnet liefert das perfekt — schneller, gleich gut.
Beispiel 2 · Hub-Code bauen
Bau mir eine neue Factsheet-Sektion im Hub mit Drag-and-Drop Staffing.
→ OPUS. Komplexer Code mit vielen Abhängigkeiten, State-Management, und Edge Cases. Hier zahlt sich Opus aus — Sonnet macht mehr Fehler bei großen Code-Bases.
Beispiel 3 · Quick Check während Event
Was ist der Unterschied zwischen GEMA und AKM?
→ HAIKU. Faktenfrage, kurze Antwort, kein Reasoning nötig. In 3 Sekunden fertig statt 15.
Beispiel 4 · Promoter-Performance Analyse
Analysiere die letzten 12 Events von Toni vs. Sasha — Revenue, Kosten, Auslastung, Trend.
→ SONNET (oder Opus wenn Daten sehr komplex). Analyse-Task mit Daten aus MAIN DB. Sonnet ist stark genug und deutlich schneller als Opus für strukturierte Datenanalyse.
Beispiel 5 · Daily Briefing
Was steht heute an? Check Kalender, Mails, offene Tasks.
→ SONNET. Multi-Tool-Task (Calendar + Gmail + Drive MCPs), aber kein komplexes Reasoning. Sonnet handled das perfekt, schneller als Opus.

Die Entscheidungs-Regel in 10 Sekunden

Task kommt rein │ ▼ Ist es ein einfacher Fakt / kurze Übersetzung / Quick-Check? │ JA ──────────────────────────────► HAIKU ⚡ │ NEIN │ ▼ Ist es Code-Architektur / Strategie / hat Sonnet bereits versagt / sehr langer komplexer Kontext? │ JA ──────────────────────────────► OPUS 🧠 │ NEIN │ ▼ SONNET ✅ (Default)
Sektion 4

🔄 Modell wechseln — So geht's

In claude.ai (Web/Mobile)

Oben im Chat-Interface siehst du den Modell-Selector (aktuell steht da vermutlich "Claude Sonnet 4.6"). Klick drauf → Dropdown → wählen.

💡 Wichtig

Du kannst das Modell mitten in einem Gespräch wechseln. Der Kontext bleibt erhalten. Wenn Sonnet bei einem Code-Problem feststeckt, wechsel zu Opus — das Gespräch läuft nahtlos weiter.

Wann mid-conversation wechseln?

  • Sonnet → Opus: Wenn Sonnet den gleichen Fehler zweimal macht oder ein komplexes Reasoning-Problem auftaucht
  • Opus → Sonnet: Nach komplexem Code-Build — für Follow-up-Fragen und Erklärungen reicht Sonnet
  • → Haiku: Wenn du nach einem langen Chat nur noch eine kurze Faktenfrage hast

Im API / Automation-Kontext

Wenn du später Automationen baust (Modul 13+), setzt du das Modell per model-Parameter:

model: "claude-opus-4-6"      // Tiefe Analyse
model: "claude-sonnet-4-6"   // Default
model: "claude-haiku-4-5-20251001"  // Speed
Sektion 5

🚨 Anti-Patterns — Was du vermeiden sollst

Anti-Pattern #1
Opus für alles

Du nutzt immer Opus weil "der ist ja am besten". Ergebnis: langsamer, und kein echter Qualitäts-Unterschied bei 80% der Tasks. Sonnet ist für Daily Operations gleichwertig — nur schneller.

Anti-Pattern #2
Haiku für komplexe Tasks

Du nutzt Haiku weil es "schnell genug" ist — für einen komplexen Security-Order mit 4 Events und 3 Zeitslots. Haiku übersieht Details, macht Logik-Fehler. Für alles was Präzision braucht: Sonnet mindestens.

Anti-Pattern #3
Nicht wechseln wenn's nicht klappt

Sonnet macht zum dritten Mal den gleichen Fehler beim Hub-Code. Du probierst weiter mit Sonnet. Falsch — das ist das Signal für Opus. Modell-Wechsel ist kein Versagen, sondern richtiges Tool-Selection.

Anti-Pattern #4
Modell-Wechsel = neuer Chat

Du glaubst du musst einen neuen Chat starten um das Modell zu wechseln. Nein — du wechselst mid-conversation. Kontext bleibt, Modell ändert sich. Kein Reset nötig.

Hands-on Übung

🛠️ Üben — 15 Minuten

Übung · Modell-Entscheidung
5 Tasks, 5 Entscheidungen

Geh in dein Turbo-Projects Projekt. Mach für jeden der folgenden Tasks bewusst eine Modell-Entscheidung bevor du anfängst:

  1. Task: "Schreib einen kurzen Insta-Caption für das nächste Turbo-Event." → Welches Modell?
  2. Task: "Analysiere meine letzten 6 EIGEN-Events nach Revenue-Trend." → Welches Modell?
  3. Task: "Was bedeutet 'Headliner-Fee' auf einer Bookingrechnung?" → Welches Modell?
  4. Task: "Bau mir einen neuen Revenue-Calculator als Hub-Sektion mit Live-Berechnung." → Welches Modell?
  5. Task: "Draft eine formelle Mail an Sandra wegen Renovierungszeitplan." → Welches Modell?

Lösung: 1=Sonnet, 2=Sonnet, 3=Haiku, 4=Opus, 5=Sonnet
